1. Перша нормальна форма (1NF)
Питання: Чи є поля, що містять списки або множинні значення?
Відповідь: Ні.
У всіх таблицях кожне поле містить атомарні (неділимі) значення:
Немає полів типу VARCHAR, де зберігаються списки через кому ("tag1,tag2,tag3").
Немає полів-масивів.
Всі дані рознесені по окремих таблицях (borrowings і reviews).

Висновок: Схема знаходиться у 1NF.
2. Друга нормальна форма (2NF)
Питання: Чи є атрибути, що залежать не від усього первинного ключа (актуально при складених ключах)?
Відповідь: Ні.
У всіх таблицях використовується простій сурогатний первинний ключ (id serial / BIGSERIAL).
Оскільки первинний ключ складається з одного поля, автоматично виконується 2NF (немає часткової залежності).
Висновок: Схема знаходиться у 2NF.

3. Третя нормальна форма (3NF)
Питання: Чи є атрибути, що transitively залежать один від одного, а не безпосередньо від первинного ключа?
Відповідь: Ні.
Аналіз таблиць:
У таблиці users поле email є унікальним, але не залежить від інших нефункціональних полів.
У таблиці books поля title, author, isbn залежать тільки від id книги.
У таблиці borrowings дати (borrow_date, due_date, return_date) і status залежать від конкретної видачі (від id).
Немає транзитивних залежностей типу «zip_code → city», «book_category → category_name» тощо.

# Документація бази даних Library System

## Опис таблиць

### 1. Таблиця `users`
**Призначення:** Зберігає інформацію про користувачів системи (читачі та адміністратори).  
**Основні поля:**
- `id` — унікальний ідентифікатор
- `email` — email-адреса (унікальне)
- `password_hash` — хеш пароля
- `first_name`, `last_name` — ім'я та прізвище
- `role` — роль користувача (`USER` / `ADMIN`)
- `created_at` — дата реєстрації

### 2. Таблиця `books`
**Призначення:** Каталог книг бібліотеки.  
**Основні поля:**
- `id` — унікальний ідентифікатор книги
- `title` — назва книги
- `author` — автор
- `isbn` — унікальний ISBN
- `description` — опис книги
- `available` — доступність книги (true/false)
- `created_at` — дата додавання

### 3. Таблиця `borrowings`
**Призначення:** Фіксує факт видачі книги користувачу.  
**Основні поля:**
- `id` — ідентифікатор видачі
- `book_id` — посилання на книгу
- `user_id` — посилання на користувача
- `borrow_date` — дата видачі
- `due_date` — дата повернення
- `return_date` — фактична дата повернення
- `status` — статус (`ACTIVE`, `RETURNED`, `OVERDUE`)

### 4. Таблиця `reviews`
**Призначення:** Зберігає відгуки користувачів про книги.  
**Основні поля:**
- `id` — ідентифікатор відгуку
- `book_id` — книга
- `user_id` — автор відгуку
- `rating` — оцінка від 1 до 5
- `comment` — текст відгуку
- `created_at` — дата створення

## Ключові зв'язки

- Один користувач може взяти багато книг (`users` → `borrowings` — 1:N)
- Одна книга може видаватися багато разів (`books` → `borrowings` — 1:N)
- Одна книга може мати багато відгуків (`books` → `reviews` — 1:N)
- Один користувач може залишити багато відгуків (`users` → `reviews` — 1:N)

Всі зв'язки реалізовані через **Foreign Key**.
