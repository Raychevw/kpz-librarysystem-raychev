# ADR-001: Вибір архітектурного стилю

## Статус
Прийнято

## Контекст
Розробляється система обліку книг бібліотеки (Library System) з функціями пошуку, бронювання, історії видач та адміністрування. Проєкт виконується одним студентом у рамках навчального курсу «Конструювання ПЗ».

## Рішення
Використовувати **3-шарову архітектуру (Layered Architecture)** з патерном **MVC**.

Шари:
- **Presentation Layer** — обробка HTTP-запитів, валідація
- **Business Logic Layer** — бізнес-правила
- **Data Access Layer** — абстракція доступу до БД

## Альтернативи, що розглядалися

- **Мікросервісна архітектура**: відхилено, бо занадто складна для одного розробника
- **Event-Driven Architecture**: відхилено, бо в системі немає потреби в асинхронній обробці
- **Clean Architecture**: відхилено через надмірну абстракцію для невеликого проєкту

## Наслідки

**Переваги (+)**:
- + Простота реалізації та розуміння
- + Чіткий розподіл обов'язків
- + Легко замінити БД
- + Легко тестувати кожен шар окремо

**Недоліки (-)**:
- - Ускладнюється додавання нових типів клієнтів
- - Business Logic Layer може стати «божественним сервісом»
