# kpz-librarysystem-raychev

## Опис
Library System — це застосунок для управління бібліотекою, який дозволяє зручно вести облік книг, користувачів і процесу видачі та повернення літератури. Система автоматизує основні операції, спрощує пошук книг і допомагає контролювати доступність ресурсів.

## Технології
- Мова програмування: Python 3.12
- Фреймворк: FastAPI
- База даних: PostgreSQL
- ORM: SQLAlchemy  # (або Tortoise ORM – виберіть один)

## Як запустити
1. Встановіть Python 3.12.
2. Клонуйте репозиторій: `git clone https://github.com/Raychevw/kpz-librarysystem-raychev.git`
3. Встановіть залежності: `pip install -r requirements.txt`
4. Налаштуйте змінні середовища у файлі `.env`.
5. Запустіть сервер: `uvicorn src.main:app --reload`

## Architecture Decisions / Рішення щодо якості коду
SOLID принципи
У проєкті застосовано принципи SOLID для покращення архітектури та підтримуваності коду.
- SRP (Single Responsibility Principle)
Кожен модуль виконує одну роль: репозиторії працюють з базою даних, сервіси містять бізнес-логіку, моделі описують дані.
- DRY (Don't Repeat Yourself)
Повторювана логіка винесена в окремі функції (наприклад, валідація), щоб уникнути дублювання коду.
- DIP (Dependency Inversion Principle)
Залежності передаються через конструктори, що дозволяє замінювати реалізації без зміни бізнес-логіки.
- OCP (Open/Closed Principle)
Система може розширюватися новими модулями без зміни існуючого коду.
- KISS (Keep It Simple, Stupid)
Логіка розділена на прості частини без перевантажених методів.

Організація структури модулів
Проєкт побудований за шаровою архітектурою:
- models — сутності (книги, користувачі, видачі)
- repositories — робота з базою даних
- services — бізнес-логіка
- utils — допоміжні функції та валідація

Уникнуті Code Smells
- God Object — великі класи розділені на модулі
- Duplicate Code — повторювана логіка винесена у функції
- Hard Dependencies — залежності передаються через конструктори
- Long Methods — складні методи розбиті
- Magic Numbers — замінені константами
